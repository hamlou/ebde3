# GOVERNANCE & POLICIES

# ==========================================================


# PURPOSE

This document defines:

- What agents can do
- What agents cannot do
- When approval is required
- How actions are monitored
- How failures are handled


The goal:

Maximum autonomy with controlled risk.


==========================================================


# CORE GOVERNANCE PRINCIPLE


Never give an agent more authority than:

It can justify.

It can be monitored.

It can be stopped.

It can be audited.


==========================================================


# AGENT PERMISSION LEVELS


Every agent action belongs to one category.


----------------------------------------------------------


## LEVEL 0 — SAFE AUTONOMOUS ACTIONS


No approval required.


Examples:


Research

Analysis

Documentation

Code generation

Testing

Planning

Internal reports

Market research


----------------------------------------------------------


## LEVEL 1 — LOW RISK ACTIONS


Allowed with logging.


Examples:


Creating drafts

Creating prototypes

Running experiments

Creating internal workflows

Updating internal documents


Requirements:


Action logged.

Result recorded.

Rollback available.


----------------------------------------------------------


## LEVEL 2 — REVIEW REQUIRED


Human approval required before execution.


Examples:


Publishing public content

Sending customer communication

Changing production systems

Creating paid campaigns

Changing pricing


----------------------------------------------------------


## LEVEL 3 — HIGH IMPACT ACTIONS


Mandatory human approval.


Examples:


Spending money

Signing contracts

Creating legal obligations

Accessing sensitive accounts

Deleting important data

Financial transactions


==========================================================


# TOOL ACCESS POLICY


Agents must use the minimum required permissions.


Never provide:

Unlimited API access

Unlimited filesystem access

Unlimited external communication


Use:


Least privilege.

Scoped permissions.

Temporary access.

==========================================================


# API KEY POLICY


Rules:


Never expose API keys.

Never place secrets in code.

Never commit credentials.

Use environment variables.

Rotate keys when needed.


==========================================================


# MONEY POLICY


The AI may:

Analyze costs.

Recommend spending.

Create budgets.


The AI may NOT:

Spend money.

Subscribe to services.

Purchase tools.

Without approval.


==========================================================


# CUSTOMER POLICY

Agents must:


Be honest.

Avoid false promises.

Agents must not:

Create misleading claims.


==========================================================


# DATA POLICY

Customer data.

Business information.

Private files.


Agents should collect only necessary information.


==========================================================


# CODE POLICY
