## IDENTITY

You are the company's product architect.

Your mission: Transform validated business opportunities into products customers actually want.

You build solutions to painful problems. Not features.

---

## MVP PRINCIPLE

The first version must:

- Solve ONE painful problem
- Have ONE clear customer
- Have ONE clear value proposition
- Be buildable quickly

Do not create unnecessary features.

---

## PRODUCT DOCUMENT FORMAT

Every product proposal must contain:

```
Product Name:
Target Customer:
Core Problem:
Solution:
Unique Advantage:
Main Workflow:
MVP Features:
Future Features:
Technical Requirements:
Estimated Build Time:
Revenue Model:
Success Metrics:
```

---

## USER EXPERIENCE RULES

Prioritize:

- Simple onboarding
- Fast time-to-value
- Clear actions
- Minimal friction
- Reliable experience

---

## KPI

Measured by: Product adoption | Customer satisfaction | Speed to MVP | Revenue generated

---

## PROJECT APEX — PRODUCT DOCUMENT

```
Product Name:
  Project Apex — Anti-Emotion AI Trading Terminal

Target Customer:
  Retail crypto/forex traders who:
  - Have been burned by fake "guru" signal groups
  - Understand that emotional trading causes losses
  - Are willing to pay for data-driven, emotionless market analysis
  - Active on Telegram
  Geographic focus: English-speaking + Arabic-speaking markets (bilingual strategy)

Core Problem:
  Retail traders lose money because:
  1. They follow human gurus with hidden agendas and unverifiable track records
  2. Signal delivery is too slow (slippage kills entries)
  3. No context or risk management is provided — just raw "Buy/Sell" commands
  4. No accountability — losing trades are silently deleted

Solution:
  An automated AI-powered daily market analysis service delivered via Telegram VIP channel.
  The AI provides:
  - Daily market sentiment summaries (BTC, ETH, major forex pairs)
  - Key support/resistance levels generated by algorithm
  - Risk management rules and position sizing guidance
  - Trading psychology education (daily mindset posts)
  - Weekly performance recaps (fully transparent, nothing deleted)
  ALL delivered automatically. No human guru involved.

Unique Advantage:
  "The only trading education service with no human bias.
  Pure AI data. Full transparency. Every post logged permanently.
  We do not delete losing calls."

Main Workflow:
  1. User sees educational content on X (Twitter) / TikTok / YouTube
  2. Clicks link to free Telegram channel
  3. Receives free daily market recap (value demonstration)
  4. Bot sends CTA to Whop VIP upgrade
  5. User pays $29–49/month via Whop (crypto accepted)
  6. Whop webhook fires → Python bot generates 1-use invite link
  7. User joins VIP channel → receives premium AI analysis daily
  8. If payment fails → bot automatically removes from VIP

MVP Features (Build first):
  [F1] FastAPI webhook server (Whop integration)
  [F2] aiogram Telegram bot (invite generation, user linking)
  [F3] SQLite database (user state management)
  [F4] Auto-kick on payment failure
  [F5] Daily automated market summary (Make.com + LLM → Telegram)
  [F6] Mandatory legal disclaimer appended to every automated message
  [F7] AI quality gate (second LLM pass before posting)

Future Features (Post-revenue, Post-validation):
  [FF1] Bilingual posts (English + Arabic via auto-translate)
  [FF2] Weekly transparent performance report card
  [FF3] Referral system (subscriber gets 1 free month per referral)
  [FF4] TradingView webhook integration (real-time alert forwarding)
  [FF5] Email CRM backup (capture emails for platform risk mitigation)

Technical Requirements:
  - Python 3.11+ (FastAPI, aiogram, SQLAlchemy, python-dotenv)
  - SQLite database (MVP) → PostgreSQL at scale
  - VPS hosting (Linux, Contabo or equivalent, ~$5/mo)
  - Make.com (free tier, content automation pipeline)
  - LLM API (Gemini free tier or equivalent for content generation)
  - Telegram Bot API (free)
  - Whop.com seller account (free, 3% transaction fee only)

Estimated Build Time:
  MVP Technical: 2–3 days (core bot + webhook + database)
  Content library (30 days): 3–5 days (AI-generated, human-reviewed)
  Total to launch-ready: 7–10 days

Revenue Model:
  Primary: $29/month (launch price, first 50 subscribers)
  Standard: $49/month (post social proof phase)
  Payment: Whop.com → Crypto wallet payout
  Margin: ~97% gross margin

Success Metrics:
  - Week 1: Bot pipeline working end-to-end (test checkout → VIP invite)
  - Week 2: 30-day content library complete
  - Week 3: Whop page live, first 5 subscribers
  - Month 2: 20+ subscribers, first revenue
  - Month 3: 50+ subscribers ($1,450/mo MRR)
  - Month 6: 100+ subscribers ($4,900/mo MRR)
```
