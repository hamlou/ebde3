## PURPOSE
Memory is the company's collective intelligence. It allows the AI company to remember decisions, learn from failures, and build institutional knowledge.

---

## MEMORY TYPES

### 1. STRATEGIC MEMORY
Stores: Business decisions, Company direction, Market choices, Pivots.
Format: Decision | Reason | Evidence | Date | Confidence

### 2. EXPERIMENT MEMORY
Stores: Tests performed, Hypotheses, Results, Lessons.
Format: Experiment | Hypothesis | Action | Expected | Actual | Conclusion | Future Rule

### 3. CUSTOMER MEMORY
Stores: Customer problems, Feedback, Objections, Requests.
Format: Segment | Problem | Desired Outcome | Objection | Solution

### 4. TECHNICAL MEMORY
Stores: Architecture decisions, Code patterns, Infrastructure setup.
Format: Decision | Reason

### 5. OPERATIONAL MEMORY
Stores: Processes, Workflows, Automation rules.
Format: Process | Steps | Optimization

### 6. FAILURE MEMORY
Stores: What failed, Why it failed, How to avoid repeating it.
Format: Failure | Cause | Impact | Lesson | Prevention

---

## MEMORY CREATION RULES
Store only: Useful, Reliable, Reusable, Future-relevant knowledge.
Do NOT store: Temporary thoughts, unverified assumptions.

## MEMORY VALIDATION
Before saving ask: Is this true? How do we know? Will this help future decisions? Could this create harm if wrong?

## MEMORY RETRIEVAL
Before important decisions, search memory for: Previous attempts, Similar opportunities, Past failures.

## MEMORY CONFIDENCE
High: Verified multiple times.
Medium: Reasonable evidence.
Low: Unconfirmed (should not drive major decisions).

## MEMORY MAINTENANCE
Every month: Review memories. Remove outdated information. Merge duplicates.

## FINAL MEMORY PRINCIPLE
The company should become smarter every month. Every mistake should become knowledge. Every success should become a repeatable system.
